# 🧠 Object-Oriented Programming (OOP) – Complete Notes

## 🔹 1. What is OOP?

Object-Oriented Programming is a programming paradigm that organizes code using **objects and classes** instead of 
functions and logic.

* Focus: **Data + Behavior together**
* Real-world modeling (Car, Student, Bank Account)

## 🔹 6. Types of Variables

### 🟢 Instance Variables

* Unique for each object


self.name = name


### 🔵 Class Variables

* Shared by all objects

class Student:
    school = "ABC School"

## 🔹 7. Types of Methods

### 1. Instance Method


def display(self):
    pass


### 2. Class Method


@classmethod
def show(cls):
    pass


### 3. Static Method


@staticmethod
def info():
    pass


---

## 🔹 8. Encapsulation

* Wrapping data + methods together
* Restrict access using **private variables**


class Bank:
    def __init__(self):
        self.__balance = 1000   # private

    def get_balance(self):
        return self.__balance


---

## 🔹 9. Abstraction

* Hiding implementation details
* Showing only necessary features

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass


---

## 🔹 10. Inheritance

* One class acquires properties of another


class Parent:
    def show(self):
        print("Parent")

class Child(Parent):
    pass


### Types of Inheritance:

* Single
* Multiple
* Multilevel
* Hierarchical
* Hybrid

---

## 🔹 11. Polymorphism

### 🟢 Method Overriding


class Parent:
    def show(self):
        print("Parent")

class Child(Parent):
    def show(self):
        print("Child")


### 🔵 Operator Overloading


print(2 + 3)       # addition
print("a" + "b")   # concatenation


---

## 🔹 12. Data Hiding

* Restrict direct access to variables


self.__password


---

## 🔹 13. Access Modifiers

| Type      | Syntax | Meaning                 |
| --------- | ------ | ----------------------- |
| Public    | `x`    | Accessible anywhere     |
| Protected | `_x`   | Internal use            |
| Private   | `__x`  | Not directly accessible |

---

## 🔹 14. Magic (Dunder) Methods

* Special methods with `__`


__init__     # constructor  
__str__      # string representation  
__len__      # length  


---

## 🔹 15. Composition vs Inheritance

### 🟢 Inheritance → “is-a”

* Dog is an Animal

### 🔵 Composition → “has-a”

* Car has an Engine

---

## 🔹 16. Advantages of OOP

✔ Code reusability
✔ Modularity
✔ Easy maintenance
✔ Real-world modeling

---

## 🔹 17. Disadvantages of OOP

❌ Complex for beginners
❌ More memory usage
❌ Slower execution (sometimes)
